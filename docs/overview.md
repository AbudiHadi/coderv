## What this is

The **CoderLap Docs Toolkit** вЂ” 5 slash commands for Claude Code that help devs keep clear docs without getting tired. This repo is what users `git clone`; a one-liner installer (`curl -fsSL https://coderv.dev/install.sh | bash`) copies the skills into `~/.claude/skills/` where Claude Code discovers them. <!-- src: install.sh -->

Marketing site + docs live in the separate [`coderv-docs`](https://coderv.dev) repo.

## The 5 commands

| Command | Role |
|---|---|
| `/docify` | Once per project вЂ” scans the codebase, writes `CLAUDE.md` + reference docs with source citations. <!-- src: skills/docify/SKILL.md --> |
| `/before` | Pre-code checklist вЂ” reads docs, checks past decisions, states a plan, waits for approval. <!-- src: skills/before/SKILL.md --> |
| `/decision` | Log an ADR in 30 seconds while the choice is fresh. <!-- src: skills/decision/SKILL.md --> |
| `/ship` | Pre-commit checklist вЂ” auto-updates docs, validates citations, drafts commit message. <!-- src: skills/ship/SKILL.md --> |
| `/session` | End-of-session handoff. `/session last` reads the previous one. <!-- src: skills/session/SKILL.md --> |

## How the installer works

`install.sh` does three things: <!-- src: install.sh:1-100 -->

1. Detects its own repo path via `SCRIPT_DIR`.
2. Copies every folder from `skills/` into `~/.claude/skills/` (or `./.claude/skills/` with `--project`).
3. Appends the marker `<!-- claude-docs-toolkit -->` to each copied `SKILL.md`. This marker is what `--uninstall` looks for вЂ” so our skills can be removed cleanly without touching unrelated skills with the same name.

Flags: `--force` (overwrite), `--uninstall` (remove our skills), `--project` (install into CWD, not home).
